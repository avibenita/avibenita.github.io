📋 DATA FLOW (8 STEPS)
─────────────────────────────────────────────────────────────────────────────

  1. JavaScript: User clicks AI button
     └─> AIButton.init() → sendToVB6('RequestAIInterpretation', jsonData)

  2. VB6: Message Handler (A_HTML_Panels.frm)
     └─> Case "RequestAIInterpretation"
         └─> A_IlamaRespones.Show
         └─> A_llamaAIintegration.HandleHypothesisAIRequest(jsonData)

  3. VB6: AI Module (0llamaAIintegration.bas)
     └─> HandleHypothesisAIRequest()
         └─> Stores jsonData in pendingTestData (module variable)
         └─> Shows A_IlamaRespones form

  4. VB6: Popup Form (A_IlamaRespones.frm)
     └─> Form_Load → OrdoWebView1.init
     └─> OrdoWebView1_InitComplete → Loads ai-loading.html (spinner)
     └─> OrdoWebView1_DocumentComplete → Triggers ProcessPendingAIRequest()

  5. VB6: Process Request (0llamaAIintegration.bas)
     └─> ProcessPendingAIRequest()
         └─> Parse JSON → Dictionary
         └─> BuildHypothesisTestPrompt() → Creates AI prompt
         └─> SendRequestForHypothesis() → Calls Groq API

  6. External: Groq API
     └─> Model: llama-3.3-70b-versatile
     └─> Temperature: 0.7, Max tokens: 500
     └─> Returns: AI interpretation text

  7. VB6: Generate HTML (0llamaAIintegration.bas)
     └─> SaveResponseAsHTML()
         └─> Creates styled HTML file with interpretation
         └─> Saves to: HTML_AI_response\hypothesis-interpretation.html
         └─> Calls A_IlamaRespones.ShowResponse()

  8. VB6: Display Response (A_IlamaRespones.frm)
     └─> ShowResponse()
         └─> Navigates OrdoWebView1 to generated HTML
         └─> User sees interpretation in popup

🎯 KEY CODE SNIPPETS
─────────────────────────────────────────────────────────────────────────────

Message Handler (A_HTML_Panels.frm):
  Case "RequestAIInterpretation"
    A_IlamaRespones.Show
    A_llamaAIintegration.HandleHypothesisAIRequest sMsgContent

Store Data (0llamaAIintegration.bas):
  Public Sub HandleHypothesisAIRequest(ByVal jsonTestData As String)
    pendingTestData = jsonTestData  ' Store in module variable
    A_IlamaRespones.Show            ' Show popup form
  End Sub

Process AI Request:
  Public Sub ProcessPendingAIRequest()
    Set testDict = JsonConverter.ParseJson(pendingTestData)
    prompt = BuildHypothesisTestPrompt(testDict)
    aiResponse = SendRequestForHypothesis(prompt)
    Call SaveResponseAsHTML(aiResponse, "HTML_AI_response\hypothesis-interpretation.html")
  End Sub

Map Test Orientation:
  Select Case testOrientation
    Case "Two-sided"
      orientationText = "Two-Tailed Test"
    Case "Right-tailed", "One-sided"
      orientationText = "One-Tailed Test (Right-Tailed)"
    Case "Left-tailed"
      orientationText = "One-Tailed Test (Left-Tailed)"
  End Select

API Call:
  Set http = CreateObject("MSXML2.XMLHTTP.6.0")
  http.Open "POST", "https://api.groq.com/openai/v1/chat/completions", False
  http.setRequestHeader "Authorization", "Bearer " & API_KEY
  http.Send payload
  ' Parse response and extract AI text

Prevent Duplicate Processing:
  Private Sub OrdoWebView1_DocumentComplete()
    If Not isProcessing Then
      isProcessing = True
      A_llamaAIintegration.ProcessPendingAIRequest
    End If
  End Sub

🐛 COMMON ISSUES
─────────────────────────────────────────────────────────────────────────────

❌ "No pending test data to process"
  → Cause: pendingTestData is empty
  → Fix: Ensure data stored BEFORE showing form

❌ Spinner spins endlessly
  → Cause: DocumentComplete fires multiple times
  → Fix: Use isProcessing flag to prevent re-entry

❌ API Error 401
  → Cause: Invalid API key
  → Fix: Update API_KEY constant in 0llamaAIintegration.bas

❌ Wrong test type (one-tailed shows as two-tailed)
  → Cause: testOrientation not mapped correctly
  → Fix: Update Select Case in BuildHypothesisTestPrompt()

❌ HTML doesn't display
  → Cause: File path incorrect or WebView not initialized
  → Fix: Check Debug.Print output for file path
